# AI / docs artifact policy

gridgram uses a single-source-of-truth (SSOT) model for everything agents and
docs consume. The rules below apply unconditionally.

## Never hand-edit these files

They are regenerated by scripts; any manual edit will be overwritten:

- `src/generated/**` — icon-index.json, llm-reference.md
- `docs/public/llms.txt`
- `docs/public/llms-full.txt`
- `docs/public/examples/**` (built by `scripts/build-docs-examples.ts`)

The `rules` array inside `context7.json` is also a generated concern in
spirit — edit only after reading the "Why" below. Other fields in
`context7.json` (projectTitle, folders, excludeFolders, …) are hand-maintained.

## If you want to change what agents see, edit the source

| Want to change…                              | Edit this                                                |
| -------------------------------------------- | -------------------------------------------------------- |
| `.gg` grammar / tokens                       | `src/gg/dsl.ts` (its leading BNF comment is the SSOT)    |
| CLI flag surface / help text                 | `src/cli/args.ts` and `src/cli/commands/*.ts`            |
| Icon tags / aliases                          | `src/data/icon-tags.json` (optional overrides)           |
| Tabler icon data itself                      | `bun run sync-tabler` after upgrading `@tabler/icons`    |
| LLM-reference prose / examples               | `src/templates/llm-reference.template.md`, `examples/`   |
| Human-facing guide content                   | `docs/en/**` (and `docs/ja/**` for localized pages)      |

After any of those edits, regenerate with `/regen-ai` (see the regen-ai
skill) or run `bun run ai:regen` directly.

## Why

Duplicated content drifts. The CLI help, the docs CLI reference, the LLM
bundle, and the llms.txt link list all needed the same information; keeping
five copies by hand was the failure mode. Now the source is in one place
(the code / templates / examples) and everything else is derived.
